### 1.2 CloseButton Component Update

**File**: `src/components/shared/CloseButton.tsx`

**Changes**:

1. `rounded-lg` вҶ’ `rounded-full` (11/12 modals use rounded-full)
2. **Add optional `ariaLabel` prop** вҖ” different targets use different labels:
   - Modals/dialogs: `common.closeDialog` (this is the DEFAULT, no override needed)
   - Sheets/sidebars: `common.close` (4 targets: WeightQuickLog, SetEditor, DishManager, CalendarTab вҖ” override required)
   - Special: `imageCapture.closeCamera` (1 target: ImageCapture вҖ” override required)
3. **Add `variant` prop** вҖ” `'default'` for card/modal surfaces, `'overlay'` for camera/dark surfaces (ImageCapture only)
4. Default aria-label: `common.closeDialog` (backward compatible with current code, majority use case)

**Final component**:

```tsx
interface CloseButtonProps {
  onClick: () => void;
  'data-testid'?: string;
  /** Resolved aria-label string. Caller passes t('...') result. Default: t('common.closeDialog') computed internally. */
  ariaLabel?: string;
  /** Visual variant. 'default' = card/modal surface. 'overlay' = dark/video surface (white icon, backdrop-blur). */
  variant?: 'default' | 'overlay';
}

const VARIANT_STYLES = {
  default: 'text-muted-foreground hover:text-foreground hover:bg-accent focus-visible:ring-ring',
  overlay: 'text-white bg-card/20 hover:bg-card/30 backdrop-blur focus-visible:ring-ring',
} as const;

export const CloseButton = ({
  onClick,
  'data-testid': testId,
  ariaLabel,
  variant = 'default',
}: Readonly<CloseButtonProps>) => {
  const { t } = useTranslation();
  return (
    <button
      type="button"
      onClick={onClick}
      data-testid={testId}
      aria-label={ariaLabel ?? t('common.closeDialog')}
      className={`flex min-h-11 min-w-11 items-center justify-center rounded-full transition-all focus-visible:ring-2 focus-visible:outline-none ${VARIANT_STYLES[variant]}`}
    >
      <X className="h-5 w-5" />
    </button>
  );
};
```

**Variants**:

- `default`: Standard card/modal surface вҖ” muted icon, hover accent background
- `overlay`: Camera/video/dark surface вҖ” white icon, translucent backdrop-blur background. Used only by ImageCapture.

**ariaLabel mapping per target**:
| Label | Targets |

| `t('common.closeDialog')` **(DEFAULT вҖ” no override needed)** | ClearPlanModal, CopyPlanModal, TemplateManager, SaveTemplateModal, MealPlannerModal, DishEditModal, IngredientEditModal, QuickAddIngredientForm, AISuggestionPreviewModal, AISuggestIngredientsPreview (Г—2), SaveAnalyzedDishModal, DetailModal |

### 2.4 CombinedHero.tsx (NEW вҖ” ~80 LOC)

**Purpose**: Orchestrate NutritionSection + WeeklyStatsRow within a single hero card. Owns root container, error isolation, loading/first-time state coordination, and calls `useDailyScore()` for cross-cutting state.

**Types**: Import `ScoreColor` from `@/features/dashboard/types` and `HeroContext` from `@/features/dashboard/hooks/useDailyScore`.

```tsx
interface CombinedHeroProps {
  isLoading?: boolean;
}

export const CombinedHero = memo(({ isLoading }: Readonly<CombinedHeroProps>) => {
  const { t } = useTranslation();

  // CombinedHero calls useDailyScore to get cross-cutting state
  const { isFirstTimeUser, greeting, heroContext, totalScore, color } = useDailyScore();

  // Derive localized score label from ScoreColor: emeraldвҶ’excellent, amberвҶ’good, slateвҶ’needsWork
  const SCORE_COLOR_TO_LABEL: Record<ScoreColor, string> = {
    emerald: 'excellent',
    amber: 'good',
    slate: 'needsWork',
  };
  const scoreLabel = t(`dashboard.hero.scoreLabel.${SCORE_COLOR_TO_LABEL[color]}`);

  return (
    <section
      className="from-primary to-primary/90 text-primary-foreground rounded-2xl bg-gradient-to-br p-4 shadow-sm"
      aria-label={t('dashboard.hero.a11yLabel', { score: totalScore, label: scoreLabel })}
      aria-busy={isLoading}
    >
      {/* Top: Greeting + Nutrition вҖ” receives score data as props */}
      <NutritionSection
        isLoading={isLoading}
        isFirstTimeUser={isFirstTimeUser}
        greeting={greeting}
        heroContext={heroContext}
        totalScore={totalScore}
        scoreColor={color}
      />

      {/* Divider вҖ” hidden when WeeklyStatsRow is hidden */}
      {!isFirstTimeUser && <div className="border-primary-foreground/10 my-3 border-t" />}

      {/* Bottom: Weekly Stats Row вҖ” own ErrorBoundary for fault isolation */}
      {!isFirstTimeUser && (
        <ErrorBoundary fallback={<WeeklyStatsRowFallback />}>
          <WeeklyStatsRow />
        </ErrorBoundary>
      )}
    </section>
  );
});
CombinedHero.displayName = 'CombinedHero';
```

**Data flow**: CombinedHero is the ONLY component that calls `useDailyScore()`. It passes results as props to NutritionSection. NutritionSection does NOT call useDailyScore itself вҖ” it's a "presentation + nutrition data" component.

**Error isolation**: `WeeklyStatsRow` wrapped in its own `ErrorBoundary` so that a failure in weekly stats (store read error, NaN, etc.) does NOT take down the entire hero. The `<WeeklyStatsRowFallback />` renders the 3-col grid structure with "вҖ”" placeholders.

**Prerequisite**: `ErrorBoundary` must be extended to accept `fallback?: ReactNode` prop (see В§3 Step 0).

**Loading state**: When `isLoading=true`:

- NutritionSection shows skeleton (same as current NutritionHero behavior)
- WeeklyStatsRow renders normally (it reads from persistent Zustand store, not async data)

**First-time user state**: When `isFirstTimeUser=true` (setup incomplete вҖ” `!allSetupComplete` requires non-default health profile AND at least one meal AND at least one logged workout):

- NutritionSection shows setup prompt (same as current NutritionHero behavior)
- WeeklyStatsRow is **not rendered** вҖ” no weight/streak/adherence data exists yet, showing "0 | 0 | 0%" would be misleading
- The divider is also not rendered

Moving from Tier 3 (lazy, deferred) to Tier 1 (immediate) means this computation runs on every initial render. For users with large datasets (400+ weight entries from `src/__tests__/integration/dashboardEdgeCases.test.ts` scenario), ensure computations are memoized or selector-derived.
