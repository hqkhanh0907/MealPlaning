User mở app buổi sáng, chưa ăn gì, chưa tập. Daily Score phải fair (redistribute weights).

📝 Morning-specific behaviors:
• Daily Score: Hiện score + "(cập nhật khi log thêm)" label
• Energy Balance: "0 / 1500 kcal" — progress bar 0%, "Log bữa sáng" CTA
• Today's Plan: Full training card + meals "0/3 bữa"
• Insight: P8 Tip (no trigger conditions met yet)
• Quick Actions: Center = "➕ Log bữa sáng"

🥗 State 4: Nutrition-Only User (No Training Plan)

User chỉ dùng meal tracking, không tạo training plan. Dashboard phải adapt, không ép tập.
🧮 Daily Score Adjustment (No Fitness):
Factor Normal Nutrition-Only
🍽️ Calories 30% 40%
🥩 Protein 25% 30%
🏋️ Workout 25% 0% (excluded)
⚖️ Weight 10% 15%
🔥 Streak 10% 15%
Rule: Nếu training_plans table trống → redistribute workout weight
📱 UI Differences:
• Today's Plan card: Chỉ hiện meals side (full width), ẩn workout side
• Quick Actions: Ẩn "🏋️ Bắt đầu tập", thay bằng "📊 Macro detail" hoặc "📸 Scan food"
• Insight P2: Protein suggestion vẫn hoạt động bình thường
• Insight P4-P5: Streak/PR workout insights bị skip → fallback P6/P7/P8
• Gentle upsell: Sau 14 ngày sử dụng, hiện 1 lần: "💪 Kết hợp tập luyện giúp tối ưu kết quả — Khám phá"

🔄 State 5: Goal Transition (Cut → Bulk → Maintain)

User thay đổi mục tiêu. Dashboard phải acknowledge transition, không silent switch.
🔄 Transition Handling:
🔄
Mục tiêu đã thay đổi!
Cutting (−500 kcal) → Lean Bulk (+300 kcal)
Target mới: 2,100 kcal • 166g protein
Khi user đổi goal trong Settings:
1. Hiện transition banner ở T4 (thay insight) trong 3 ngày
2. Reset Daily Score calculation với target mới ngay lập tức
3. Weight trend color logic tự adapt (Cut→Bulk: gaining = green)
4. Energy Balance target update tự động
5. Streak KHÔNG reset — continuity across goals
6. Sau 3 ngày, banner tự dismiss → normal insight

📴 State 6: Error & Offline Handling

🛡️ Resilience Rules:
SQLite failure (rare):
• Hiện cached data từ last successful read + "(dữ liệu cũ)" label
• Silent retry mỗi 30s, max 3 retries, sau đó hiện "Đang gặp sự cố — kéo xuống để thử lại"

Component-level failure:
• Nếu 1 component fail (ví dụ: weight_log query lỗi), chỉ component đó hiện skeleton
• Các component khác vẫn hoạt động bình thường — KHÔNG cascade failure
• Mỗi component có try/catch riêng, fallback = compact skeleton + "Thử lại"

Offline mode (no network):
• 100% functional — vì dùng local SQLite, không cần network cho dashboard
• Chỉ ảnh hưởng nếu user dùng Google Drive sync → hiện "📴 Offline — data chưa đồng bộ" ở top

Performance budget:
• Dashboard load target: <200ms (8 queries song song)
• Mỗi SQL query target: <25ms
• Lazy load T4-T5 (AI Insight + Quick Actions) sau khi T1-T3 render
