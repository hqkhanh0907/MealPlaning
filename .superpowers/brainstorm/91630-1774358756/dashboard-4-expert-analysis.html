👔 1. Business Analyst / Product Manager — KPIs & User Pain Points

🎯 Core Question: "Người dùng cần nhìn thấy gì để biết hành trình sức khỏe đang đi đúng hướng?"
Ứng dụng này phục vụ 3 nhóm mục tiêu: Giảm cân (Cut), Tăng cân (Bulk), Duy trì (Maintain).
Dashboard phải trả lời được 5 câu hỏi then chốt trong < 5 giây.
Q1: "Hôm nay tôi đang ở đâu?"
→ KPI: Energy Balance (Calories IN − OUT vs Target)
REAL-TIME
Q2: "Tôi có đang tiến bộ không?"
→ KPI: Weight Trend (7-day moving avg direction) + Volume Trend
WEEKLY
Q3: "Tôi có kiên trì không?"
→ KPI: Adherence Rate (% ngày đạt calo/protein) + Streak
DAILY
Q4: "Có vấn đề gì cần điều chỉnh?"
→ KPI: Auto-Adjust Alerts + Plateau Detection + Overtraining Warning
14-DAY
Q5: "Tiếp theo tôi cần làm gì?"
→ KPI: Next Action (workout hôm nay, bữa ăn thiếu, log cân nặng)
NOW

🎨 2. UI/UX Designer — Information Hierarchy & Visual Treatment

📐 Nguyên tắc: Information Hierarchy theo F-Pattern
Mobile users scan theo pattern Z (top-left → top-right → bottom-left → bottom-right).
Thông tin quan trọng nhất ở top 1/3 screen. CTAs ở thumb zone (bottom half).
Visual Hierarchy — 5 tầng thông tin
TẦNG 1 — Glance (0.5s)
🎯 Daily Score + Greeting
Tổng quan 1 con số — "Hôm nay bạn đạt 82/100"
Font: 28-36px bold | Color: White on gradient | Space: ~80px height
TẦNG 2 — Scan (2s)
⚡ Energy Balance + 🥩 Protein
2 metrics quan trọng nhất hôm nay — progress bars visual
Font: 16-20px | Color: Emerald/Blue | Space: ~100px
TẦNG 3 — Read (5s)
📋 Today's Plan + ⚖️ Weight + 🔥 Streak
Cards grid 2×2 — mỗi card 1 thông tin đọc nhanh
Font: 12-14px | Color: Neutral + accent | Space: ~120px
TẦNG 4 — Analyze (10s)
💡 AI Insight + ⚠️ Alerts
Insight text + warning banners (nếu có)
Font: 11-12px | Color: Amber/Red accent | Space: ~60px
TẦNG 5 — Act (optional)
🔗 Quick Actions (log weight, start workout, log meal)
Sticky bottom area trong thumb zone
Font: 11px | Color: Emerald buttons | Space: ~50px
Tổng: ~410px — vừa 1 screen iPhone SE (568px viewport) → Zero scroll cho core info

🧠 Tâm lý học hành vi áp dụng:
Loss Aversion: Streak counter tạo động lực "không muốn mất" → hiệu quả hơn reward

📈 3. Data Analyst / BI Specialist — Charts, Aggregations & Filters

📊 Nguyên tắc: "Right chart for right data"
Mỗi loại dữ liệu cần loại biểu đồ phù hợp. Sai biểu đồ = misleading data = sai quyết định.
📊 Ma trận lựa chọn biểu đồ cho Dashboard:
Data Point Loại dữ liệu Chart Type Lý do Vị trí
Energy Balance Part-of-whole (today) Stacked Progress Bar So sánh IN vs OUT vs remaining — linear intuitive Dashboard Tầng 2
Macro Split Proportion (3 parts) Donut Chart (SVG) 3 thành phần P/C/F — reuse MacroChart hiện có Tap Energy → Detail
Weight Trend Time series Line Chart (dual) Daily dots + 7-day moving avg line — show noise vs trend Tap Weight card → Detail
Volume Trend Time series (weekly) Bar Chart (vertical) So sánh volume tuần này vs tuần trước — discrete values Fitness Progress
Calorie vs Target Comparison (daily) Dual Bar Chart Actual (green/red) vs target (gray) — visual gap Tap Energy → History
Adherence % Single value + trend Ring Gauge + Sparkline Percentage → ring (0-100%); trend → sparkline mini Dashboard Tầng 3
Protein progress Progress (today) Linear Progress Bar current/target → simple linear bar (120/166g) Dashboard Tầng 2
Streak Counter + timeline Number + Dot Array Big number (🔥12) + 7-day dots (✓/😴/📍) Dashboard Tầng 3
1RM Estimates Multi-value comparison Horizontal Bar Chart Squat/Bench/Deadlift/OHP — so sánh các lift Fitness Progress
Muscle Group Balance Distribution Radar/Spider Chart 6 nhóm cơ — thấy imbalance ngay lập tức Fitness Progress
🔢 Công thức tính toán (Aggregations):
Daily Score (0-100)
score = (calorie_adherence × 0.30) + (protein_adherence × 0.25) + (workout_completed × 0.25) + (weight_logged × 0.10) + (streak_bonus × 0.10)
calorie_adherence: 100 nếu ±100kcal, giảm tuyến tính | workout_completed: 100 nếu tập/nghỉ đúng plan | streak_bonus: min(streak_days × 2, 100)
Weekly Adherence Rate
adherence_cal = COUNT(days WHERE |actual - target| ≤ 100) / 7 × 100%
adherence_protein = COUNT(days WHERE |actual_protein - target_protein| ≤ 10g) / 7 × 100%
Weight Moving Average
moving_avg = SUM(weight_log.weight_kg WHERE date >= today-7) / COUNT(*)
trend = current_7d_avg - previous_7d_avg // >0 = gaining, <0 = losing
🔍 Filter Strategy — khi nào dùng filter nào:
Filter Áp dụng cho Mục đích
Hôm nay Energy Balance, Protein, Score Real-time tracking (Dashboard main)
1W (7 ngày) Adherence, Weight avg, Streak Short-term consistency check
1M (30 ngày) Weight trend, Volume trend, Calorie trend See if plan is working
3M / Cycle Full cycle review, 1RM progress Phase evaluation
All time Weight history, Total sessions, PRs Journey overview
Note: Dashboard main view = "Hôm nay" only. Filters chỉ available khi tap card → expand detail view.

⚙️ 4. Data Engineer — Data Pipeline & SQL Queries

🔧 Architecture: Local-first SQLite Pipeline
Không có server/API — tất cả data nằm trong SQLite local (sql.js WASM trên web, Capacitor SQLite trên mobile).
Dashboard queries chạy trực tiếp trên client → phải tối ưu cho mobile devices.
🔄 Data Flow Pipeline:
📥 Input Layer
User actions
Log meal / workout / weight
→
⚙️ Compute Layer
Zustand stores
Calculate metrics
→
📊 Display Layer
Dashboard UI
Charts & cards
Tables:
daily_log, workout_sets, weight_log, adjustments
Stores:
dashboardStore, fitnessStore, healthProfileStore
Components:
EnergyBalance, WeightCard, StreakCounter...
💾 Core SQL Queries cho Dashboard:
-- Q1: Today's Energy Balance
SELECT target_calories, actual_calories, target_protein, actual_protein
FROM daily_log WHERE date = DATE('now');
-- Q2: Today's Exercise Calories
SELECT SUM(estimated_calories) as total_burned
FROM workout_sets ws JOIN workouts w ON ws.workout_id = w.id
WHERE w.date = DATE('now');
-- Q3: 7-day Weight Moving Average
SELECT AVG(weight_kg) as moving_avg,
  MAX(weight_kg) - MIN(weight_kg) as fluctuation
FROM weight_log WHERE date >= DATE('now', '-7 days');
-- Q4: Weekly Adherence (Cal + Protein)
SELECT
  ROUND(AVG(adherence_cal) * 100, 1) as cal_adherence_pct,
  ROUND(AVG(adherence_protein) * 100, 1) as protein_adherence_pct
FROM daily_log WHERE date >= DATE('now', '-7 days');
-- Q5: Streak Count (consecutive plan-adherent days)
-- Computed in fitnessStore (not pure SQL)
-- Logic: walk backward from today, count consecutive
-- days where workout_logged OR planned_rest_day = true
-- Q6: Latest Auto-Adjustment
SELECT reason, old_target_cal, new_target_cal, date
FROM adjustments WHERE applied = 1
ORDER BY created_at DESC LIMIT 1;
⚡ Performance Optimization:
Strategy Implementation Impact
SQLite Index CREATE INDEX idx_daily_log_date ON daily_log(date) Query time: ~50ms → ~2ms
Zustand Cache dashboardStore caches today's data, invalidate on new log Avoid re-query on tab switch
Lazy Detail Charts History charts (30d+) loaded on-demand khi tap card Initial render: ~3 queries vs ~8
Batch Updates When logging meal: update daily_log + dashboardStore in 1 transaction Single re-render, not cascading
