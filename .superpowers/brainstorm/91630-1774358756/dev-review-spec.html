2️⃣ 🔴 Navigation — No Router + Full-screen Pages

BLOCKING: Spec yêu cầu "full-screen pages" cho WorkoutLogger/CardioLogger nhưng app KHÔNG có router
Hiện tại: Tab navigation bằng activeMainTab state trong App.tsx. Không có React Router, không có history stack. Spec yêu cầu: L1 → L2 → L3 navigation, full-screen pages ẩn bottom nav, "← Quay lại" predictable, floating mini indicator khi rời tab.
3 Options:

A. Giữ không dùng Router (hiện tại):
• Dùng state stack: navigationStack: string[] trong Zustand
• Push/pop pattern cho full-screen pages
• ⚠️ Rủi ro: Back button Android phức tạp, deep linking impossible, history bị mất khi refresh

B. Thêm React Router (recommended):
• react-router-dom v7 (compatible với Vite 6)
• Routes: /calendar, /fitness, /fitness/workout/:id, /dashboard
• Bottom nav auto-hide via route matching
• ✅ Back button native, deep linking, history persistence
• ⚠️ Migration effort: refactor all tab switching logic

C. Lightweight custom router:
• Zustand-based navigation store with history stack
• Interceptor cho Android back button (đã có useModalBackHandler)
• ✅ Zero new dependency, fits current architecture
• ⚠️ Reinventing the wheel, no deep linking
💡 Recommendation: Option C — Lightweight custom router
Lý do: App là mobile-first Capacitor app — deep linking ít quan trọng. Custom router dùng Zustand phù hợp hơn vì:
• Không thêm dependency mới
• Kiểm soát hoàn toàn bottom nav hide/show
• Consistent với Zustand stores pattern spec đã chọn
• Extend useModalBackHandler hiện có cho full-screen pages

3️⃣ 🔴 State Migration — localStorage → Zustand → SQLite

BLOCKING: Spec nói "Big Bang Migration" nhưng thiếu chi tiết migration path
Hiện tại: 6 state slices dùng usePersistedState (localStorage). Zustand đã install nhưng CHƯA dùng.
Spec Phase 1: localStorage → SQLite. Phase 2: Update hooks dùng DatabaseService.
Vấn đề: Spec không nói rõ Zustand đóng vai trò gì trong migration. Zustand ĐÃ nằm trong dependencies nhưng codebase chưa dùng.
Layer Hiện tại Spec Phase 1-2 Final State
Persistence localStorage SQLite (sql.js / Capacitor) SQLite
Reactive UI usePersistedState (useState wrapper) ??? Zustand stores
Cross-tab state Props drilling từ App.tsx ??? Zustand (global access)
💡 Đề xuất: Migration Path rõ ràng
Step 1 (Phase 0): usePersistedState → Zustand stores (vẫn localStorage backend). Zero behavior change, chỉ refactor.
Step 2 (Phase 1): Zustand stores → SQLite backend. Swap persistence layer, Zustand vẫn là reactive cache.
Step 3 (Phase 2): New features (fitness, dashboard) dùng trực tiếp SQLite + Zustand pattern từ đầu.
Result: Tất cả state đi qua: SQLite (source of truth) → Zustand (reactive cache) → React (render)
